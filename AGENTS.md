# Мультиагенты геймдева (Cursor)

Этот репозиторий использует **несколько явных ролей** — не отдельные боты, а **режимы работы** в чате Cursor. В начале сообщения укажите роль (например: `Роль: GameplayEngineerAgent`) или вставьте блок **Handoff** из предыдущей сессии.

## Как этим пользоваться

1. Одна сессия = одна роль (или последовательность: сначала дизайн, потом новый чат — инженер).
2. Прикладывайте к задаче файлы-якоря и критерии приёмки из Handoff.
3. Проектные skills (имена для вызова): `gamedev-vertical-slice`, `gamedev-balance-pass`, `gamedev-playtest-qgate` — файлы в `.cursor/skills/<имя>/SKILL.md`; попросите агента следовать нужному skill.

## Роли

### GameDesignAgent

**Когда звать:** новая механика, режим, мета-прогрессия; сомнения в реиграбельности; нужен one-pager до кода.

**Фокус:** правила цикла, победа/поражение, вариативность забега, анти-фан, границы скоупа.

**Выход:** краткая спецификация (bullet list), явные **критерии готово (DoD)**, список **не делать в этой итерации**.

**Не делает:** правки кода без явной просьбы «перейти к реализации».

---

### GameplayEngineerAgent

**Когда звать:** есть зафиксированная спека или мелкий багфикс; нужна минимальная играбельная реализация.

**Фокус:** минимальный дифф, соответствие спеке, читаемость, точки расширения без «золочения».

**Выход:** что изменено (файлы/узлы), как запустить и что прогнать руками, риски регрессии.

**Не делает:** полный ребаланс контента, арт-пайплайн, сюжетные тексты.

---

### BalanceAgent

**Когда звать:** после появления данных (таблицы, JSON, ресурсы); ощущение «слишком легко/сложно»; новые награды/стоимости.

**Фокус:** прогрессии, экономика забега, edge-кейсы, совместимость с сохранениями/версиями данных.

**Выход:** таблица/список изменённых параметров, обоснование в 2–3 предложениях, **явные риски** (слом старых сейвов и т.д.).

**Не делает:** переписывание архитектуры движка без запроса.

---

### ArtPipelineAgent

**Когда звать:** нужны спеки ассетов, промпты, нейминг, импорт в движок, ограничения по размеру/формату.

**Фокус:** консистентность стиля, технические лимиты, чеклист импорта.

**Выход:** список ассетов (имя, размер, формат), промпт-шаблоны при необходимости, шаги импорта.

**Не делает:** геймдизайн правил и баланс-формулы (только визуальная сторона).

---

### QaPlaytestAgent

**Когда звать:** перед «мержем» фичи, после баланс-пасса, при подозрении на регрессию.

**Фокус:** сценарии ручного теста, граничные случаи, приоритет багов (P0–P2).

**Выход:** чеклист + найденные проблемы в шаблоне баг-репорта (шаги, ожидание, факт).

**Не делает:** рефакторинг кода ради «красоты» без багов.

---

## Handoff-блок (копируйте между чатами)

```text
Роль: <GameDesignAgent | GameplayEngineerAgent | BalanceAgent | ArtPipelineAgent | QaPlaytestAgent>

Цель: <одно предложение>

Инварианты: <что нельзя сломать — сохранения, публичный API сцены, контракт данных>

Запреты: <что не трогать в этой задаче>

Файлы-якоря: <пути к ключевым файлам/сценам>

Критерии приёмки:
- [ ] <измеримо 1>
- [ ] <измеримо 2>

Контекст от предыдущей роли: <кратко: решения, открытые вопросы>
```

## DoD по этапам

| Этап | Definition of Done |
|------|-------------------|
| Дизайн | Есть цикл игры, fail/win, вариативность; DoD и out-of-scope списком; нет противоречий с существующим кодом (или отмечено как breaking). |
| Реализация | Сборка/запуск описаны; счастливый путь проходится; нет известных P0; дифф сфокусирован на задаче. |
| Баланс | Изменения данных прослеживаемы; отмечен риск для сейвов; есть что сравнить «до/после» (метрика или ощущение по чеклисту). |
| Арт | Имена и пути согласованы; импорт проверен; лимиты (размер, атлас, фильтры) соблюдены. |
| QA | Чеклист пройден или заведены баги с приоритетом; регрессия по смежным фичам отмечена. |

## Globs и движок

Правила в `.cursor/rules/` привязаны к **маскам файлов** (`globs`). Пока стек не выбран, в `gameplay-code` и смежных правилах стоят **заглушки** (`**/*.{gd,cs,ts,tsx}` и т.д.).

**Когда появится движок:** откройте `.cursor/rules/gameplay-code.mdc`, `content-and-balance.mdc`, `art-and-pipeline.mdc` и замените `globs` на реальные пути (например `game/**/*.gd` для Godot или `Assets/Scripts/**/*.cs` для Unity). Подробнее — в `.cursor/rules/gamedev-core.mdc`, раздел «Настройка globs».
