# Пиксельная UI-дизайн система

## Цель

Единая тёплая пиксельная система интерфейса для всех UI-слоёв игры:

- диалоги
- выборы
- интро
- карточная сцена
- deck view
- бой
- концовки и титры

Цвета задаются существующими токенами из [`COLOR_SYSTEM.md`](./COLOR_SYSTEM.md) и [`../src/theme/tokens.css`](../src/theme/tokens.css). Эта система добавляет не новую палитру, а правила структуры и применения.

## Foundation

Токены находятся в [`../src/theme/tokens.css`](../src/theme/tokens.css).

Основные группы:

- `--ui-font-body`, `--ui-font-display`
- `--ui-space-*`
- `--ui-border-*`
- `--ui-radius-pixel`
- `--ui-shadow-pixel`
- `--ui-panel-*`
- `--ui-button-*`
- `--ui-chip-*`
- `--ui-overlay-scrim`

## Паттерны

Общие классы находятся в [`../src/style.css`](../src/style.css).

### Overlay

- `.pixel-overlay`
- `.pixel-overlay--center`
- `.pixel-overlay--bottom`

Использовать для всех полноэкранных UI-слоёв над канвасом.

### Panel

- `.pixel-panel`
- `.pixel-panel--strong`

Использовать для диалогов, choice-box, intro-box, deck-view-box, end-screen-box, card-scene-shell и ключевых боевых панелей.

### Typography

- `.pixel-title`
- `.pixel-subtitle`
- `.pixel-text`

Если экрану нужен собственный класс, он должен сохранять тот же ритм: uppercase-заголовок, плотные рамки, короткие интервалы.

### Buttons

- `.pixel-button`
- `.pixel-button--ghost`
- `.pixel-button--accent`

Все новые кнопки UI должны опираться на эти паттерны.

### Supporting

- `.pixel-chip`
- `.pixel-list`
- `.pixel-stack`
- `.pixel-divider`

## Правила применения

### Диалоги и выборы

- Текстовая панель снизу или по центру.
- Одна основная рамка.
- Никаких стеклянных фонов, мягких радиусов и web-card тени.

### Интро и концовки

- Те же панели и кнопки, что в диалогах.
- Различаться должны только размером блока и плотностью текста.

### Карточная сцена

- Карточки могут быть декоративнее, чем диалоги, но должны использовать те же panel/button token values.

### Бой

- Бой остаётся fullscreen.
- У боя допустим более плотный HUD и более высокий контраст.
- Но рамки, кнопки, панели и статусные элементы должны быть частью той же системы, а не отдельного визуального мира.

## Темы

Темы управляются через `html[data-game-state]` и [`../src/theme/syncGameTheme.ts`](../src/theme/syncGameTheme.ts).

Система должна сохранять:

- одинаковую структуру компонентов между темами
- разные цвета и акценты через токены
- читаемость во всех состояниях `surface`, `anxiety`, `calm`, `finale`

## Анти-паттерны

Не добавлять без причины:

- большие радиусы
- blur / glassmorphism
- случайные локальные HEX вместо токенов
- отдельные шрифтовые системы на каждый экран
- отдельную кнопку/панельную стилистику для одного режима, если это можно выразить через общие паттерны
